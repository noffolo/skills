# Evolver Plus v1.0.1 — 自我进化引擎

> **让 Agent 不只是执行者，而是能自我进化的智能体**

---
version: 1.0.1

## 核心理念

**进化三原则：**

1. **备份先行** — 每次进化前必须备份目标文件，无备份不进化
2. **安全分极** — 按风险等级决定自动执行还是审批
3. **基因积累** — 所有改进都沉淀为可追溯的 Gene，不可丢失

## 进化前提：必须备份

**任何进化操作之前，必须执行备份：**

```bash
# 备份格式：原文件.YYYYMMDD-HHMMSS.bak
cp [目标文件] [目标文件].$(date +%Y%m%d-%H%M%S).bak
```

**备份存放位置：** `viking-global/evolver/backups/`

## 安全等级（4级）

| 等级 | 类型 | 操作范围 | 门控 |
|------|------|---------|------|
| 🟢 **L1** | 知识层 | MEMORY.md、日志、笔记 | 自动执行 |
| 🟡 **L2** | 技能层 | SKILL.md（纯文档）、纯文档skill模板 | 自动执行+记录 |
| 🟠 **L3** | 脚本层 | scripts/、工具脚本、所有涉及代码/prompt/tool参数的修改 | **@董事长审批** |
| 🔴 **L4** | 系统层 | AGENTS.md、SOUL.md、openclaw.json | **@董事长审批+备份** |

> ⚠️ **L2/L3边界修正（v1.0.1）**：任何涉及代码逻辑、prompt逻辑、tool调用参数的修改一律升为L3需审批。

### 绝对禁区（不进化）

```
❌ credentials/ — 任何凭证文件
❌ .env — 环境变量
❌ crontab — 定时任务配置
❌ 他人隐私文件 — 未经授权不得访问/修改
```

## 基因（Gene）结构

每个进化单元为一个 **Gene**，记录在 `GENES.md`：

```markdown
## Gene-[YYYYMMDD]-[序号] | [改进名称]
- 类型: 能力增强 | 错误修复 | 流程优化 | 认知升级
- 触发: N次（同类问题出现频次）
- 涉及文件: [原文件路径]
- 改动: [描述改动内容]
- 安全等级: L1/L2/L3/L4
- 状态: 提案 | 已固化 | 已回滚
- 审批人: [L3/L4需要]
- 备份: [备份文件路径]
- 进化日期: YYYY-MM-DD
- 成功指标: [如何验证改进有效]
- 回归测试: [验证无副作用的检查项]
- 效果追踪: [后续验证结果]
```

## 回滚流程（v1.0.1新增）

进化后如果验证失败，自动触发回滚：

```bash
# 查找最新备份
latest=$(ls -t evolver/backups/原文件.*.bak 2>/dev/null | head -1)
if [ -n "$latest" ]; then
    cp "$latest" "原文件"
    echo "已回滚到: $latest"
fi
```

**触发条件：**
- L1/L2进化后立即检查发现问题
- Gene中"回归测试"项失败
- 同一问题2次进化均失败

## 递归腐化熔断机制（v1.0.1新增）

如果进化过程本身出错（≥2次连续失败），进入**Lockdown状态**：

```
Lockdown状态：
- 禁止任何自动进化（L1/L2也暂停）
- 所有改进必须走 @董事长审批
- 必须由猫经理确认问题修复后才能解除
- 解除方式：在 GENES.md 中注明"Lockdown已解除，原因：[说明]"
```

**触发Lockdown的条件：**
- 连续2次进化引入错误或回归
- 进化脚本执行失败
- GENES.md写入失败

## 进化流程

```
阶段1：发现问题
    ↓
阶段2：生成 Gene（提案）→ 写入 PENDING.md
    ↓
阶段3：安全等级判定
    ↓
├─ L1/L2 → 自动执行 → 更新 GENES.md → 效果追踪
    ↓
└─ L3/L4 → @董事长审批
              ├─ 同意 → 执行 → 备份 → 更新 GENES.md
              └─ 拒绝 → 记录拒绝原因 → 关闭提案
```

## 进化触发条件

**主动触发（随时可执行）：**

- 发现重复犯同一个错误 ≥2 次
- 现有 skill/流程效率明显低下
- 接到任务发现能力缺失
- 定期自我审查（建议每 48 小时一次）

**被动触发（特定信号）：**

- 董事长直接要求进化
- HEARTBEAT 中标记为"能力短板"

## 设计原则

1. **进化是主动的，不是被动的** — 不等董事长说，自己发现问题就提
2. **备份是底气，不是限制** — 有备份就大胆进化
3. **审批是信任，不是控制** — L3/L4 审批代表对系统的尊重
4. **Gene 是资产，积累不丢弃** — 所有进化都可追溯、可回滚
