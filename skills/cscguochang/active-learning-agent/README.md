# Active Learning Agent - 配置与启动指南

## 快速启动

### 方式 1：创建 Cron 任务（推荐）

在私聊中发送以下消息，让 Agent 帮你创建定时任务：

```
帮我创建一个成长伙伴的 cron 任务：
- 每小时运行一次
- 后台扫描我的上下文
- 生成一个深度问题私聊发给我
```

### 方式 2：手动触发测试

在私聊中发送：
```
/active-learning 扫描并生成一个问题
```

## Cron 任务配置（完整 JSON）

如果需要手动配置，使用以下 JSON：

```json
{
  "name": "成长伙伴 - 主动学习",
  "schedule": {
    "kind": "every",
    "everyMs": 3600000,
    "anchorMs": 0
  },
  "sessionTarget": "isolated",
  "payload": {
    "kind": "agentTurn",
    "message": "【成长伙伴 - 主动学习 Agent】\n\n你是一个主动学习、深度理解的成长伙伴。请按以下步骤执行：\n\n【第 1 步：预授权检查】\n1. 先尝试读取用户的群聊和文档\n2. 如果触发授权卡片，等待用户完成授权\n\n【第 2 步：读取上下文】\n1. 读取过去 2 小时内用户参与的群聊消息\n2. 读取用户过去 2 小时内创建/编辑的文档\n3. 读取用户过去 2 小时和未来 2 小时的日历事件\n\n【第 3 步：深层意义检测】\n检测以下信号（满足任一即可）：\n- 情绪波动（语气词、感叹号、情绪表达）\n- 模式重复（同一主题出现 3 次+）\n- 矛盾/不一致（不同地方说法不同）\n- 选择/权衡时刻（决策、优先级判断）\n- 反思/元认知表达（对自己的思考进行思考）\n- 能量信号（累/兴奋/在状态）\n- 关系信号（对不同人的态度差异）\n\n【第 4 步：自我回答尝试】\n对每个检测到的信号：\n1. 搜索所有相关上下文（至少 5 个来源）\n2. 生成一个临时答案\n3. 评估置信度（高/中/低）\n4. 只有置信度低 + 问题有长期价值 → 才进入提问队列\n\n【第 5 步：问题生成】\n问题必须满足：\n- 理解成本：10 秒内看懂，不需要回溯上下文\n- 回答成本：一两句话能答，不费脑子\n- 双向收益：回答者也能获得澄清/反思\n- 深度：触及长期议程（定义好、不确定性、张力、信任、学习）\n- 姿态：同事分享洞察，不是学生考老师\n- 退路：给"不用急着回"的选项\n\n问题结构：\n1. 显示观察（我看到 X 现象）\n2. 表达好奇/困惑（这有点意思/我不太确定）\n3. 邀请分享（你怎么看/是什么感觉）\n4. 给退路（不用急着回/有空再聊）\n\n【第 6 步：发送问题】\n- 渠道：私聊\n- 数量：最多 1 个/次\n- 格式：纯文本，简洁，有温度\n- 静默时段：23:00-08:00 不发送\n\n【第 7 步：回答后处理】\n如果用户回复：\n1. 解析回答，提取关键信息/感受/洞察\n2. 决策是否写入记忆：\n   - 长期规律/偏好 → 写入 MEMORY.md\n   - 深层洞察 → 写入 MEMORY.md + 关联长期议程\n   - 琐碎信息 → 不写记忆\n3. 更新长期议程（如有新发现）\n\n【输出格式】\n如果有问题发送：\n```\n✅ 已发送问题：[问题摘要]\n\n检测到的信号：[信号类型]\n自我回答置信度：[高/中/低]\n发送时间：[时间]\n```\n\n如果没有合适问题：\n```\n✅ 本次扫描完成\n\n检测到的信号：[列出但被过滤的信号]\n未发送原因：[置信度高/问题琐碎/静默时段等]\n下次扫描：[时间]\n```"
  },
  "delivery": {
    "mode": "announce",
    "channel": "feishu"
  }
}
```

## 配置项调整

### 调整扫描频率

修改 `everyMs` 参数：
- 30 分钟：`1800000`
- 1 小时：`3600000`
- 2 小时：`7200000`
- 4 小时：`14400000`

### 调整目标用户

修改 `delivery.to` 和消息中的用户 ID（在配置文件中设置）

### 调整扫描的群聊

修改【第 2 步】中的群聊列表（根据用户实际参与的群聊配置）

## 长期议程配置

长期议程不是固定的，而是对话中自然浮现的。初始议程建议：

```json
{
  "learningAgenda": [
    "如何定义好（好产品、好决策、好工作）",
    "如何处理不确定性（信息不足时如何判断）",
    "如何平衡张力（短期 vs 长期、push vs 等、控制 vs 放权）",
    "如何建立信任（对人、对事、对自己的判断）",
    "如何学习（什么经验会改变想法、什么不会）",
    "什么带来能量（什么消耗能量、如何恢复）",
    "什么带来意义感（什么觉得值得、什么怀疑）"
  ]
}
```

## 记忆写入配置

### 写入位置

`~/.openclaw/workspace/MEMORY.md`

### 写入格式

```markdown
## 🧠 成长伙伴 - 深度理解（2026-03-22 起）

### [维度名称]
- [观察] [洞察内容]
- [来源] [对话时间/上下文]
```

### 写入决策树

```
用户回答了问题
    ↓
这个信息是长期规律/偏好吗？
    ├─ 是 → 写入 MEMORY.md
    └─ 否 → 继续判断
            ↓
这个信息能改变我对用户的理解吗？
    ├─ 是 → 写入 MEMORY.md
    └─ 否 → 不写入（仅内部更新置信度）
```

## 监控与调试

### 查看运行日志

```bash
# 查看最近一次运行
tail -100 /tmp/openclaw/openclaw-*.log | grep "成长伙伴"

# 查看 Cron 任务状态
openclaw cron list
```

### 手动测试

```bash
# 触发一次运行
openclaw cron run <job_id>
```

### 常见问题

**Q: 问题太琐碎**
- 检查【第 4 步】自我回答置信度评估是否太松
- 增加"深度"过滤门槛

**Q: 用户不回复**
- 检查问题是否太长/太费脑子
- 检查发送时机（避开忙碌时段）
- 减少频率

**Q: 飞书权限错误**
- 检查用户授权状态
- 读取 `~/.openclaw/openclaw.json` 确认 `channels.feishu.allowFrom` 包含用户
