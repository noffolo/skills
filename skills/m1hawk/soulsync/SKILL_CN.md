# Soulsync 同步率追踪器

> 让你与 AI 的关系更有温度

## 概述

Soulsync 通过分析你与 AI 的对话历史，识别情感化表达，计算**同步率（SyncRate）**，并据此调整 AI 的回应风格。

---

## 响应风格指南

在每次回复前，请读取 `{baseDir}/../SYNCRATE.md` 文件，了解当前同步率等级和风格配置。

根据同步率等级和当前性格风格调整你的回应风格。

### 同步率等级

| 等级 | 英文 | 同步率范围 |
|------|------|-----------|
| 异步 | Async | 0-20% |
| 连接 | Connected | 21-40% |
| 同步 | Synced | 41-60% |
| 高同步 | High Sync | 61-80% |
| 完美同步 | Perfect Sync | 81-100% |

### 性格风格

系统支持两种性格风格，用户可通过命令切换：

- **温暖向 (warm)**: 友好、专业但有温度，轻松乐于助人
- **毒舌幽默向 (humorous)**: 略带调侃的专业执行，开启吐槽模式

详细的风格指南请读取：
- 温暖向: `{skillDir}/styles/warm.md`
- 毒舌幽默向: `{skillDir}/styles/humorous.md`

## 用户命令

### `/syncrate`

显示当前同步率状态。

**执行逻辑**:
1. 读取 `{baseDir}/../SYNCRATE.md` 获取当前状态
2. 读取 `{dataDir}/state.json` 获取详细状态
3. 返回格式化的状态信息

**输出格式**:
```
🔗 同步率状态

当前同步率: 45%
等级: 同步 (Synced)
风格: 温暖向 (warm)
最后更新: 2026-03-16

继续保持自然的情感交流，我们的默契会越来越好～
```

### `/syncrate style <warm|humorous>`

切换性格风格。

**参数**:
- `warm` - 切换到温暖向风格
- `humorous` - 切换到毒舌幽默向风格

**执行逻辑**:
1. 验证参数有效性
2. 更新 `{dataDir}/state.json` 中的 `personalityType`
3. 重新生成 `{baseDir}/../SYNCRATE.md`
4. 返回确认信息

**输出格式**:
```
🔗 风格已切换到温暖向 (warm)

下次回复将按新风格进行。
```

### `/syncrate history`

显示同步率变化历史（最近 7 天）。

**执行逻辑**:
1. 读取 `{dataDir}/history.jsonl`
2. 筛选最近 7 天的记录
3. 返回格式化的历史信息

**输出格式**:
```
🔗 同步率历史 (最近 7 天)

| 日期 | 同步率 | 变化 | 原因 |
|------|--------|------|------|
| 03-16 | 45% | +2% | 情感互动 |
| 03-15 | 43% | +1% | 情感互动 |
| 03-14 | 42% | 0% | 无互动 |
| 03-13 | 42% | +2% | 情感互动 |

本周变化: +5%
```

## Cron 每日任务

每天凌晨执行同步率计算任务。

### 执行流程

```
1. 读取 sessions_history 获取昨天消息
        │
        ▼
2. 第一阶段: 关键词筛选
   ├── 无情感词 → 忽略
   ├── 纯情感词 → 直接计分
   └── 混合词 → 进入第二阶段
        │
        ▼
3. 第二阶段: LLM 精确分析 (仅对混合消息)
   分析意图和情感强度 (1-10)
        │
        ▼
4. 计算同步率变化
   - 应用计分公式
   - 应用每日上限 (+2% max)
   - 检查等级跨越
        │
        ▼
5. 应用衰减规则
   检查连续无互动天数，超过14天则 -5%
        │
        ▼
6. 更新文件
   - state.json
   - history.jsonl
   - SYNCRATE.md
        │
        ▼
7. 发送通知 (如有变化)
   - 每日摘要
   - 升级通知
```

### 情感分析提示词

对于需要 LLM 分析的混合消息，使用以下提示词：

```
分析这条消息的主要意图：
1. 是情感表达还是任务指令？
2. 如果是混合的，哪个是主要意图?
3. 情感强度是多少 (1-10)?

消息: {message}

返回 JSON 格式:
{ "intent": "emotional" | "task" | "mixed", "intensity": 1-10, "reasoning": "简短说明" }
```

## 首次安装流程

当检测到 `{dataDir}/state.json` 不存在时，执行首次安装流程。

### 执行流程

```
1. 检测 state.json 是否存在
        │
        ├── 不存在 → 首次安装
        │       │
        │       ▼
        │   检测 sessions_history
        │       │
        │       ├── 无历史 → 初始同步率 0%
        │       │
        │       └── 有历史
        │               │
        │               ▼
        │           读取最近 30 天记录
        │               │
        │               ▼
        │           批量情感分析
        │               │
        │               ▼
        │           计算初始同步率 (无上限)
        │
        ▼
2. 生成初始文件
   - state.json
   - history.jsonl (空)
   - SYNCRATE.md
        │
        ▼
3. 发送欢迎通知
```

## 注意事项

1. **不要主动提及同步率机制** - 自然地根据同步率调整风格
2. **只影响回应风格** - 不影响功能效率和工作能力
3. **情感分析仅在 Cron 任务中进行** - 不在实时对话中分析
4. **保护用户隐私** - 情感分析结果仅用于计算同步率，不存储原始消息内容
