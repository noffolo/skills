# 安全声明

## 技能目的
这是一个用于OpenClaw系统的任务结果推送器，用于将任务执行结果推送到华为负一屏。

## 安全特性

### 1. 无恶意行为
- 不包含恶意代码、病毒或后门
- 不进行未经授权的数据收集
- 不执行破坏性操作

### 2. 网络请求说明
- 仅向用户配置的指定URL发送HTTP POST请求
- 使用标准HTTPS协议（如果配置）
- 请求内容仅限于任务结果数据

### 3. 数据安全与本地存储

#### 数据脱敏保护
- **授权码保护**：在日志中仅显示前4位字符（如 `Twe7***`），不记录完整授权码
- **敏感信息处理**：所有敏感配置信息都进行脱敏处理
- **数据传输**：使用JSON格式，通过HTTPS协议传输

#### 本地文件存储说明
本技能会在运行过程中创建以下本地文件用于运行记录：

1. **日志文件** (`logs/` 目录)
   - **用途**：运行监控、故障排查和性能分析
   - **内容**：脱敏的运行信息、时间戳、状态码、任务元数据
   - **保留建议**：建议定期清理，可通过配置控制日志级别

2. **推送记录** (`push_records/` 目录，**可选**)
   - **用途**：历史记录、审计追踪和结果验证
   - **控制**：可通过 `save_records` 配置项关闭记录保存功能
   - **限制**：可通过 `max_records` 配置项限制最大记录数量
   - **内容**：任务响应数据、推送状态、执行时间

#### 用户控制与隐私
- **配置控制**：用户可完全控制是否保存推送记录
- **日志级别**：用户可配置日志详细程度
- **本地存储**：所有文件仅存储在用户本地设备，不外传
- **定期清理**：用户可手动清理或通过配置自动管理存储文件

### 4. 代码透明度
- 所有源代码公开可审查
- 使用标准Python库（requests, json, logging等）
- 无混淆或隐藏代码

### 5. 权限说明
- 需要网络权限：用于推送数据到负一屏服务
- 需要文件系统权限：用于记录推送历史（可选）
- 需要环境变量读取权限：用于获取配置

## 使用场景
本技能专为OpenClaw用户设计，用于：
- 将任务执行结果推送到华为负一屏
- 实现任务状态的通知功能
- 提供统一的任务结果推送接口
