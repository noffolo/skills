# 项目断档检测与恢复 (Project Resume)

## 功能概述

当用户切换聊天工具、窗口或会话后，主控能够自动检测已有项目并恢复上下文，避免因丢失上下文而重复创建项目或不知道项目进度。

## 检测时机

**主控启动时自动执行**，无需用户指令。

## 检测流程

```
主控启动
    │
    ▼
┌─────────────────────────────────────────────┐
│ Step 1：搜索项目目录                         │
│                                             │
│ 搜索路径（按顺序）：                         │
│ 1. ./projects/                            │
│ 2. ./                                      │
│                                             │
│ 查找条件：                                  │
│ - 存在 meta/metadata.json                  │
│ - 存在 meta/project.md                    │
└─────────────────┬───────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────┐
│ Step 2：读取项目状态                         │
│                                             │
│ 读取 meta/metadata.json                    │
│ 提取：                                     │
│ - project.title                           │
│ - project.version                          │
│ - project.phase                           │
│ - writingProgress.currentChapter           │
│ - writingProgress.lastCompletedChapter     │
│ - autoAdvance.status                      │
└─────────────────┬───────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────┐
│ Step 3：验证项目完整性                       │
│                                             │
│ 根据 phase 检查必要文件是否存在：            │
│                                             │
│ Phase 0-1：                                │
│   □ meta/project.md                        │

│   □ outline/outline.md                    │

                  ▼
┌─────────────────────────────────────────────┐
│ Step 4：汇报用户                            │
│                                             │

│  │                                        │

└─────────────────────────────────────────────┘
```

## Phase 对应的恢复点

| Phase | 检测到可恢复内容 | 恢复后操作 |
|-------|-----------------|-----------|
| 0 | 项目目录已创建 | 继续 Phase 0，确认设定后进入 Phase 1 |
| 1 | 世界观/角色圣经/大纲已完成 | 显示完成状态，确认后继续 Phase 1 剩余步骤或进入 Phase 2 |
| 2 | 章节已写 | 显示写作进度，可直接继续写作 |
| 3 | 维护迭代中 | 显示当前维护状态，继续维护工作 |

## 用户操作选项

### 选项1：继续当前项目
```
主控：
1. 加载项目所有必要文件到上下文
2. 根据 phase 和 currentChapter 确定恢复点
3. 询问用户想要执行的操作

示例对话：
用户：继续写作
主控：好的，继续 [项目名]。当前进度：Phase 2，第15章已完成。
请问：继续写第16章？批量写作？查看项目状态？
```

### 选项2：查看项目详情
```
主控：
1. 读取并展示 meta/project.md
2. 展示当前阶段进度
3. 列出已完成章节
4. 展示下一步建议
```

### 选项3：新建项目
```
主控：
1. 确认用户意图
2. 创建新项目目录（可与旧项目共存）
3. 开始 Phase 0
```

## 错误处理

### 情况1：检测到损坏的项目
```
主控：
"检测到项目 [名称]，但文件损坏或不完整。

损坏文件：
- meta/metadata.json (缺失)

请选择：
[修复项目] [放弃并新建项目]"
```

### 情况2：多个项目存在
```
主控：
"检测到多个项目：

1. 玄幻小说（Phase 2，已完成15章）
2. 现代悬疑（Phase 1，大纲已完成）

请选择：
[继续项目1] [继续项目2] [新建项目]"
```

## 主控启动时的完整流程

```
【主控启动】

1. 搜索项目目录
   └─ 查找 meta/metadata.json

2. 读取项目状态
   └─ 提取 phase、进度、配置

3. 检查文件完整性
   └─ 验证当前 phase 所需文件

4. 向用户汇报
   └─ 展示项目状态和选项

5. 等待用户指令
   └─ 继续/新建/查看/其他操作
```

## 铁律

- **自动检测**：主控启动时必须自动执行检测，无需用户触发
- **不自动覆盖**：检测到项目后必须用户确认才能操作
- **完整恢复**：继续项目时必须加载所有必要文件到上下文
- **状态准确**：汇报的项目状态必须与实际文件一致
